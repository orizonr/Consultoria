Sincroniza pipeline, agenda e configurações com o Supabase. Mesmo que perca o histórico do navegador, todos os dados ficam na nuvem.
🔒 Como funciona a proteção de dados
Os candidatos e vagas já ficam nativamente no Supabase — nunca se perdem. O que pode se perder ao trocar de navegador é: pipeline (kanban), agenda de entrevistas, log de auditoria e preferências. Este módulo sincroniza tudo isso com uma tabela orizon_config no Supabase, criando backup automático a cada ação importante.

Configuração da tabela no Supabase
Execute este SQL no Supabase SQL Editor para criar a tabela de configuração:
CREATE TABLE IF NOT EXISTS orizon_config ( id bigint generated always as identity primary key, chave text not null unique, valor jsonb, atualizado_em timestamptz default now() ); -- Permite leitura/escrita pela chave anon (ajuste conforme sua RLS) ALTER TABLE orizon_config ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON orizon_config FOR ALL USING (true) WITH CHECK (true);
